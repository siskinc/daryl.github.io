Golang mgo 模糊查询的使用

Thu, Jul 25, 2019

在日常使用的Mongodb中，有一项功能叫做模糊查询（使用正则匹配），例如:

db.article.find({&quot;title&quot;: {$regex: /a/, $options: &quot;im&quot;}})

这是我们常用Mongodb的命令行使用的方式，但是在mgo中做出类似的方式视乎是行不通的：

query := bson.M{&quot;title&quot;: bson.M{&quot;$regex&quot;: &quot;/a/&quot;, &quot;$options&quot;: &quot;im&quot;}}

大家用这个方式去查询，能查询到算我输！ 下面总结一下，正真使用的方式：

  1. 在Mongodb的命令行中，我们可以使用形如 \abcd\ 的方式来作为我们的pattern，但是在mgo是直接传入字符串来进行的，也就是传入的是"\a"，而不是\a\。

根据第一点，我们将代码修改一下。

query := bson.M{&quot;title&quot;: bson.M{&quot;$regex&quot;: &quot;a&quot;, &quot;$options&quot;: &quot;im&quot;}}

但是我们会发现依然不能得到我们想要的结果，那么第二点就会产生了！

  1. 在mgo中要用到模糊查询需要mgo中自带的一个结构: bson.RegEx
// RegEx represents a regular expression.  The Options field may contain

// individual characters defining the way in which the pattern should be

// applied, and must be sorted. Valid options as of this writing are &#039;i&#039; for

// case insensitive matching, &#039;m&#039; for multi-line matching, &#039;x&#039; for verbose

// mode, &#039;l&#039; to make \w, \W, and similar be locale-dependent, &#039;s&#039; for dot-all

// mode (a &#039;.&#039; matches everything), and &#039;u&#039; to make \w, \W, and similar match

// unicode. The value of the Options parameter is not verified before being

// marshaled into the BSON format.

type RegEx struct {

	Pattern string

	Options string

}

那么最终我们的代码为：

query := bson.M{&quot;title&quot;: bson.M{&quot;$regex&quot;: bson. RegEx:{Pattern:&quot;/a/&quot;, Options: &quot;im&quot;}}}
